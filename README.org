* About

Emacs control layer for the Emacs-first fork of OCP CAD Viewer:
[[https://github.com/KarimAziev/vscode-ocp-cad-viewer][KarimAziev/vscode-ocp-cad-viewer]]. This package controls the standalone
browser viewer over its websocket command API, so camera navigation can be
driven from Emacs without mouse gestures and without relying on browser focus.
For hot-reloading preview scripts, this package pairs well with [[https://github.com/KarimAziev/build123d-ocp-preview][KarimAziev/build123d-ocp-preview]].

The intended build123d workflow is:

#+begin_src text
edit build123d code in Emacs
        |
        | save
        v
build123d-ocp-preview / ocp123d
        |
        | rerun preview script
        v
vscode-ocp-cad-viewer fork
        |
        | update standalone browser viewer
        v
Emacs xwidget or browser at /viewer
        ^
        |
        | websocket camera commands
        |
ocp-cad-viewer.el
#+end_src

The three pieces have separate responsibilities:

#+begin_src text
build123d-ocp-preview   keeps geometry fresh
vscode-ocp-cad-viewer   serves/displays geometry and exposes viewer_command
ocp-cad-viewer.el       keeps camera/navigation control inside Emacs
#+end_src

* Requirements

** Emacs version
Emacs 29.1 or later, optionally built with =xwidget= support.

** Emacs libraries


| Name      | Version |
|-----------+---------|
| [[https://github.com/ahyatt/emacs-websocket][websocket]] | >=1.16  |

** External Python programs
- [[https://github.com/KarimAziev/vscode-ocp-cad-viewer][vscode-ocp-cad-viewer]]: Provides the Python =show= function. It should be installed in the virtual environment of your CadQuery/build123d project.
- [[https://github.com/KarimAziev/build123d-ocp-preview][build123d-ocp-preview]]: Provides the =ocp123d= executable, which watches project files and updates the viewer. Run it once for the same file that calls =show=, e.g.: =ocp123d -p . my_file_with_show.py= . It should also be installed in the virtual environment of your CadQuery/build123d project.

#+html: </summary>

* Usage

** Backend viewer

This package expects the forked Python viewer from
[[https://github.com/KarimAziev/vscode-ocp-cad-viewer][KarimAziev/vscode-ocp-cad-viewer]], not the upstream PyPI package. The fork adds
the ~viewer_command~ websocket protocol used by this Emacs package.

Start the standalone viewer from the Python checkout:

#+begin_src shell :eval no
cd /path/to/vscode-ocp-cad-viewer
.venv/bin/python -m ocp_vscode --port 3939
#+end_src

The viewer page is:

#+begin_src text
http://127.0.0.1:3939/viewer
#+end_src

Your CAD code can send objects to the same viewer in the normal ~ocp_vscode~
way:

#+begin_src python :eval no
import cadquery as cq
from ocp_vscode import set_port, show

set_port(3939)
show(cq.Workplane().box(1, 2, 3))
#+end_src

** Basic Emacs workflow

Configure the viewer port if you do not use the default:

#+begin_src elisp :eval no
(setq ocp-cad-viewer-port 3939)
#+end_src

Open the viewer in an xwidget/browser and connect the websocket:

#+begin_src text
M-x ocp-cad-viewer-open
#+end_src

Open a small Emacs control buffer whose keymap drives the viewer:

#+begin_src text
M-x ocp-cad-viewer-control-buffer
#+end_src

The control buffer is useful because ordinary Emacs keybindings remain active
there. The xwidget page can stay visible while the control buffer receives the
navigation commands.

You can also connect without opening the browser page:

#+begin_src text
M-x ocp-cad-viewer-connect
#+end_src

and close the websocket connection with:

#+begin_src text
M-x ocp-cad-viewer-disconnect
#+end_src

** Hot reload workflow

For build123d projects, the optional
[[https://github.com/KarimAziev/build123d-ocp-preview][build123d-ocp-preview]] package provides the ~ocp123d~ command. It starts a
long-lived ~ocp_vscode~ viewer server, watches Python files recursively, and
re-runs your preview entry script when files change.

Example:

#+begin_src shell :eval no
cd /path/to/my-build123d-project
source .venv/bin/activate
ocp123d -p . assembly.py
#+end_src

Then use this Emacs package for camera control:

#+begin_src text
M-x ocp-cad-viewer-open
M-x ocp-cad-viewer-control-buffer
#+end_src

In this setup, ~ocp123d~ keeps the geometry fresh and ~ocp-cad-viewer~ keeps
camera navigation inside Emacs.

** Keybindings

Default bindings in ~ocp-cad-viewer-mode~:

| Key         | Command                      | Effect               |
|-------------+------------------------------+----------------------|
| ~<right>~   | ~ocp-cad-viewer-rotate-z-~   | rotate right         |
| ~<left>~    | ~ocp-cad-viewer-rotate-z+~   | rotate left          |
| ~<up>~      | ~ocp-cad-viewer-rotate-x+~   | rotate up            |
| ~<down>~    | ~ocp-cad-viewer-rotate-x-~   | rotate down          |
| ~M-<left>~  | ~ocp-cad-viewer-pan-left~    | pan left             |
| ~M-<right>~ | ~ocp-cad-viewer-pan-right~   | pan right            |
| ~M-<up>~    | ~ocp-cad-viewer-pan-up~      | pan up               |
| ~M-<down>~  | ~ocp-cad-viewer-pan-down~    | pan down             |
| ~+~         | ~ocp-cad-viewer-zoom-in~     | zoom in              |
| ~-~         | ~ocp-cad-viewer-zoom-out~    | zoom out             |
| ~f~         | ~ocp-cad-viewer-view-front~  | front view           |
| ~t~         | ~ocp-cad-viewer-view-top~    | top view             |
| ~l~         | ~ocp-cad-viewer-view-left~   | left view            |
| ~r~         | ~ocp-cad-viewer-view-right~  | right view           |
| ~b~         | ~ocp-cad-viewer-view-rear~   | rear/back view       |
| ~d~         | ~ocp-cad-viewer-view-bottom~ | bottom view          |
| ~i~         | ~ocp-cad-viewer-view-iso~    | isometric view       |
| ~0~         | ~ocp-cad-viewer-reset~       | reset camera         |
| ~?~         | ~ocp-cad-viewer-menu~        | transient command UI |

Numeric prefix arguments scale movement. For example, ~C-u 4 <right>~ sends a
rotate command with four times the normal rotate step.

** Customization

Useful options:

#+begin_src elisp :eval no
(setq ocp-cad-viewer-host "127.0.0.1")
(setq ocp-cad-viewer-port 3939)
(setq ocp-cad-viewer-rotate-step 10)
(setq ocp-cad-viewer-pan-step 10)
(setq ocp-cad-viewer-zoom-step 100)
#+end_src

The command websocket connects automatically by default when a camera command
is sent:

#+begin_src elisp :eval no
(setq ocp-cad-viewer-auto-connect t)
#+end_src

** Direct commands

The package exposes direct interactive commands for fixed views, movement, and
connection management:

#+begin_src text
M-x ocp-cad-viewer-view-front
M-x ocp-cad-viewer-view-top
M-x ocp-cad-viewer-view-iso
M-x ocp-cad-viewer-reset
M-x ocp-cad-viewer-zoom-in
M-x ocp-cad-viewer-pan-left
M-x ocp-cad-viewer-rotate-x+
#+end_src

For Lisp code, use ~ocp-cad-viewer-viewer-command~ to send a protocol command
directly:

#+begin_src elisp :eval no
(ocp-cad-viewer-viewer-command "view" '((value . "front")))
(ocp-cad-viewer-viewer-command "rotate" '((axis . "x") (delta . 10)))
(ocp-cad-viewer-viewer-command "pan" '((direction . "left") (step . 10)))
(ocp-cad-viewer-viewer-command "zoom" '((delta . 100)))
#+end_src

Those calls emit websocket frames like:

#+begin_src text
C:{"type":"viewer_command","command":"view","value":"front"}
#+end_src
